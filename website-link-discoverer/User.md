# User Guide（Web 智能功能导航控制台）

这份文档面向非技术用户，按步骤说明如何安装、启动与使用前端控制台，并列出常见问题与解决方法。

---

## 1. 准备环境

> 以下命令需要在 **终端（PowerShell / CMD）** 中执行。  
> 首先将终端切换到项目目录：

```bash
cd C:\Users\zhu\Desktop\PM\Web_intelligent_function_navigation\website-link-discoverer
```

**首次使用前**，依次执行以下两条命令：

```bash
npm install
npx playwright install chromium
```

- `npm install` 会安装项目依赖，可能需要 1–3 分钟。
- `npx playwright install chromium` 会下载 Chromium 浏览器（约 150 MB），用于模拟网页访问。

> 如果下载 Chromium 失败，可以改用系统自带的 Edge 浏览器：  
> 在终端输入 `set BROWSER_CHANNEL=msedge` 后再启动服务。

---

## 2. 启动服务

确保终端已切换到 `website-link-discoverer` 目录，然后执行：

```bash
npm run server
```

看到以下字样表示启动成功：

```
Server running at http://localhost:3456
```

在浏览器地址栏输入 **`http://localhost:3456`** 即可打开控制台页面。

> ⚠️ 不要关闭终端窗口——关闭终端会停止服务。

---

## 3. 界面概览

控制台页面分为以下几个区域：

| 区域             | 作用                                                                              |
| ---------------- | --------------------------------------------------------------------------------- |
| 顶部状态栏       | 显示当前状态（就绪 / 正在爬取 / 已完成），以及"查看结果""打开报告""下载 JSON"按钮 |
| 左侧基础配置     | 输入目标网址，点击"开始爬取"或"停止"                                              |
| 高级配置（折叠） | 最大页面数、并发数、超时时间、浏览器通道、阻断资源、危险词等                      |
| 实时日志         | 黑色终端风格，实时滚动显示爬取进度                                                |
| 数据库爬取索引   | 列出 `database/` 目录下的历史结果文件，可打开报告                                 |
| 爬取结果         | 概览统计 + 页面列表 + 元素详情，支持筛选和导出                                    |

---

## 4. 开始爬取

1. 在左侧 **"目标网址"** 输入框填写要扫描的网站地址，例如 `https://www.baidu.com`。

   > 必须是完整网址，以 `https://` 或 `http://` 开头。

2. 点击绿色 **"开始爬取"** 按钮。

3. 右侧实时日志会显示浏览器启动、页面访问等进度信息；顶部状态指示灯变为"正在爬取"。

4. 爬取完成后，状态变为"已完成"，页面下方自动展示：
   - **概览**：爬取页面数、发现元素数、唯一链接数等
   - **页面列表**：每个页面的标题、URL、元素数量
   - **元素详情**：每个交互元素的类型、文本、链接、选择器

5. 如果中途想停止，点击红色 **"停止"** 按钮。

> 如需调整爬取深度、速度等参数，展开 **"高级配置"** 进行设置。  
> 不修改高级配置也可以正常使用，系统已预设了合理的默认值。

---

## 5. 查看报告

### 方式一：打开最新报告

点击顶部状态栏的 **"打开报告"** 按钮，或"数据库爬取索引"区的 **"打开最新报告"** 按钮，会新开一个报告页面，自动加载最近一次爬取的结果。

### 方式二：查看历史文件

在 **"数据库爬取索引"** 表格中，点击对应文件行的 **"查看报告"** 按钮，会在报告页中加载该历史 JSON 文件。

### 方式三：下载 JSON

点击顶部 **"下载 JSON"** 按钮，可将当前结果保存为 `.json` 文件到本地。

---

## 6. 结果文件位置

| 来源              | 保存位置                                      |
| ----------------- | --------------------------------------------- |
| Web 控制台运行    | `website-link-discoverer/outputs/`            |
| 命令行（CLI）运行 | 由 `--out` 参数指定，默认 `sitemap.json`      |
| 数据库索引        | 扫描 `website-link-discoverer/database/` 目录 |

如果你用命令行运行爬虫，把输出文件放到 `database/` 目录中，它们就会出现在控制台的索引列表里。

## 7. 常见问题

### Q: 启动报错 `EACCES: permission denied 0.0.0.0:3456`

端口被其他程序占用。解决方法：

1. 换一个端口启动：

   ```bash
   set PORT=4567
   npm run server
   ```

   然后访问 `http://localhost:4567`。

2. 或者先关闭占用端口的程序（如其他 Node 服务、IIS 等），再重新启动。

### Q: 启动报错 `Could not read package.json`

说明终端当前不在正确的目录。请先执行：

```bash
cd C:\Users\zhu\Desktop\PM\Web_intelligent_function_navigation\website-link-discoverer
```

再执行 `npm run server`。

### Q: `npm install` 失败或很慢

网络问题可能导致下载超时。尝试切换国内镜像：

```bash
npm config set registry https://registry.npmmirror.com
npm install
```

### Q: 点击"开始爬取"后没有反应 / 没有结果

请按顺序排查：

1. **服务是否在运行**——终端里是否仍在执行 `npm run server`，没有报错退出。
2. **浏览器是否已安装**——确认执行过 `npx playwright install chromium`。
3. **URL 格式是否正确**——必须以 `https://` 或 `http://` 开头，例如 `https://www.baidu.com`。
4. **网络是否可达**——公司内网或防火墙可能拦截对外请求。
5. **查看实时日志**——日志区域会显示具体的错误信息。

### Q: 报告页没有数据

- 如果是"最新报告"：确认刚完成过一次爬取，且服务没有重启（重启后内存中的结果会丢失）。
- 如果是"历史文件"：确认对应的 JSON 文件确实存在于 `website-link-discoverer/database/` 目录中。
- 可以点击"数据库爬取索引"区的 **"刷新列表"** 按钮重新扫描文件。

### Q: 如何添加历史结果到索引？

将爬虫输出的 `.json` 文件复制到 `website-link-discoverer/database/` 目录，然后点击控制台的"刷新列表"即可看到。
